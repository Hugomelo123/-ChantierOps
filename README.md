## Overview

On most construction sites, the problem is not execution.  
It is visibility.

Information exists, but:
- it arrives too late  
- it is incomplete  
- it is spread across calls, WhatsApp and memory  

As a result:
- managers react too late  
- problems grow before being seen  
- teams lose time repeating the same information  

---

## The problem

You manage several chantiers.

At the end of the day, you still do not clearly know:
- what was done  
- what is blocked  
- what is missing  

You call a team leader.

He answers:
> "ça avance"

The call ends, but nothing really changed.

The real issues (delay, safety, missing material) are often discovered the next day — when it is already too late to react properly.

---

## What this demo shows

This is **not a product**.

It is a demonstration of how I would organize field reporting inside a company:

- capture information in a simple way  
- structure it automatically  
- make it visible in real time  
- detect problems early  

Without changing how teams already work.

---

## Core idea

Instead of forcing teams to use a new app:

👉 Use WhatsApp (already used daily)  
👉 Turn simple messages into structured data  
👉 Give managers instant visibility
